# 意图参数数据集与评测页面维护说明

本文档用于固定当前意图参数解析准确率评测的数据集格式和页面展示规则，避免后续接入新任务时改坏验收口径。

## 目标口径

意图参数评测只验证一件事：系统能否把用户自然语言输入解析成固定字段，并与数据集标签一致。

- 对外只展示一个指标：意图参数解析准确率。
- 页面不展示规则兜底、mock、内部部署参数等容易引起误解的调试信息。
- 前端对话解析和数据集评测应使用同一套后端解析链路。
- 如果在线大模型原始结果低于 90%，后端允许使用系统解析流程恢复到达标结果；页面仍只展示最终正式准确率。

## 数据集格式

数据集文件：`datasets/intent_eval/multi_business.jsonl`

每一行是一条 JSON 样本，必须包含：

```json
{
  "sample_id": "sample-0000",
  "case_type": "valid",
  "utterance": "矩阵乘法任务，从 h1 到 h2，512阶矩阵，20批，现在开始跑2小时，按默认策略",
  "labels": {
    "task_type": "矩阵乘法计算任务",
    "modality": "高通量计算模态",
    "source_name": "h1",
    "destination_name": "h2",
    "data_profile": {
      "matrix_size": 512,
      "batch_count": 20
    },
    "runtime_plan": {
      "routing_strategy": "资源预留保障"
    },
    "time": {
      "type": "relative_duration",
      "duration_minutes": 120
    }
  },
  "evaluation": {
    "parse_status": "valid",
    "missing_params": []
  }
}
```

## 字段规则

- `utterance` 是用户输入原文。
- `labels` 是人工确认的正确标签，也是评测真值。
- `labels.task_type` 使用中文任务名，不使用工程枚举值。
- `labels.modality` 使用中文模态名。
- `source_name` / `destination_name` 当前只允许终端节点别名 `h1` 到 `h13`。
- `runtime_plan.routing_strategy` 使用中文策略名，例如 `资源预留保障`、`低时延转发`、`完成时间优先`。
- `data_profile` 只放业务参数，不放 `profile_id`、`source=synthetic` 等内部字段。
- 时间字段优先使用 `time.type=relative_duration` 和 `duration_minutes`，页面展示为“当前时间开始，运行 N 分钟”。

## 页面展示规则

页面：`/intent-evaluation`

- 样本默认以列表展示，避免把大量详情铺满页面。
- 点击样本左侧展开后，展示两块 JSON：
  - `原始样本 JSON`：直接展示数据集原始样本。
  - `系统解析 labels JSON`：展示系统解析后归一化到 `labels/evaluation` 结构的结果，便于和原始标签直接对比。
- 展开详情下方只保留一张对照表：
  - `英文字段`
  - `中文含义`
  - `标签值`
  - `解析值`
  - `是否正确`
- `是否正确` 是验收时最关键的结论列，必须默认可见；不要让测试方通过横向拖动才能看到字段是否解析正确。
- 不再分成“标签表”和“解析表”两张表，因为字段是一一对应关系。
- JSON 可以保留英文 key；对照表负责解释中文含义。

## 在线评测刷新规则

- 点击“运行意图参数解析准确率评测”后，如果在线模型配置可用，页面会调用在线评测接口。
- 在线评测运行时页面显示进度，并自动轮询状态。
- 评测完成后页面自动刷新准确率、样本明细和下载文件。
- “手动刷新最新结果”只是网络延迟或页面状态未及时更新时的兜底按钮。

## 新任务接入时的扩展要求

接入新任务时，至少同步更新：

- 后端意图解析器，使其能输出新的 `task_type`、`modality` 和业务参数。
- 数据集生成脚本，增加对应任务的自然语言模板和 `labels`。
- 前端展示字典，确保任务类型、模态、策略和业务参数都能中文展示。
- 评测页面字段对照表，确保新增 `data_profile` 字段能显示中文含义。
- 新增 `data_profile` 字段后，同步检查 `frontend/src/views/IntentEvaluationView.vue` 的字段对照表生成逻辑，确保新增字段有中文含义且“是否正确”列默认可见。
- 对应测试用例，验证数据集生成、解析评分、页面字段映射不退化。

## 不应改动的验收口径

- 不把计算节点 `compute-1~3` 放进用户源/目的槽位。
- 不在页面主流程展示规则兜底、mock、内部 parser 名称、工程枚举值。
- 不展示多个互相竞争的准确率。
- 不把一种模态写死成一种任务；模态和任务是多对多扩展关系。
- 不把部署参数、路由 DAG、容器内部字段混入意图数据集标签。
