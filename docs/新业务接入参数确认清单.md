# 新业务接入参数确认清单

本文用于让其他模块同学把自己的业务接入本系统。请业务提供方先按本清单补齐信息，再进入镜像适配、意图解析数据集扩展和端到端联调。

## 一、先给一句话说明

| 项目 | 请填写 |
| --- | --- |
| 业务中文名称 | 例如：视频AI推理任务、矩阵乘法计算任务 |
| 业务英文标识 | 例如：`low_latency_video_pipeline`，建议小写下划线，不要频繁改 |
| 所属模态 | 从系统八类模态中选择，例如：低时延转发模态、高通量计算模态 |
| 推荐路由策略 | 成本开销保障 / 低时延转发 / 资源预留保障 / 完成时间优先 / 资源负载均衡 |
| 业务目标 | 用一句话说明专家应该看到什么结果，例如：视频帧被识别并画框、矩阵计算返回吞吐量 |

| 参数中文名 | 参数英文键 | 类型 | 示例 | 是否必填 | 合法范围/枚举 | 说明 |

| 源节点 | `source_name` | 字符串 | `h1` | 是 | 系统拓扑节点别名 | 也可由系统根据数据面 IP 反查 |

## 三、业务容器角色

当前建议按 `source -> compute -> sink` 描述业务链路，但不同场景部署方式不同。

| 角色 | 是否需要容器 | 镜像名 | 监听端口 | 启动命令 | 主要环境变量 | 说明 |
| --- | --- | --- | --- | --- | --- | --- |
| source | 是/否 |  |  |  |  | 用户接入演示中可由用户手动启动 |
| compute | 通常是 |  |  |  |  | 平台默认调度部署的计算节点 |
| sink | 是/否 |  |  |  |  | 用户接入演示中可由用户手动启动并展示结果 |

说明：业务节点之间必须通过网络通信传递数据，不要依赖宿主机共享目录作为 source/compute/sink 的业务数据通道。MinIO 只用于结果归档和页面预览。

## 六、请提供 5-10 条自然语言样例

这些样例用于构建意图解析数据集。建议覆盖标准表达、口语表达、缺参数表达和错误节点表达。

| 样例 | 期望任务类型 | 期望模态 | 期望策略 | 期望业务参数 |
| --- | --- | --- | --- | --- |
| 从 h1 到 h2 跑视频AI推理，处理720p视频100帧，立即运行60分钟，低时延转发 | 视频AI推理任务 | 低时延转发模态 | 低时延转发 | 帧数=100，分辨率=720p |
|  |  |  |  |  |

数据集统一使用当前格式：

```json
{
  "case_type": "valid",
  "utterance": "从 h1 到 h2 跑视频AI推理，处理720p视频100帧，立即运行60分钟，低时延转发",
  "labels": {
    "task_type": "视频AI推理任务",
    "modality": "低时延转发模态",
    "source_name": "h1",
    "destination_name": "h2",
    "data_profile": {
      "frame_count": 100,
      "resolution": "720p",
      "fps": 30
    },
    "runtime_plan": {
      "routing_strategy": "低时延转发"
    },
    "time": {
      "type": "relative_duration",
      "duration_minutes": 60
    }
  },
  "evaluation": {
    "parse_status": "valid",
    "missing_params": []
  }
}
```

说明：`labels` 只写人工标注答案，也就是用户原话中应该被系统抽取出来的字段；`evaluation` 只写评测辅助信息。业务目标指标、比较符、单位等由系统根据任务类型推导，不写入人工标注答案。

## 七、接入前自查

- 业务是否能在单机 Docker 中独立跑通。
- source、compute、sink 之间是否能用 HTTP 或明确协议通信。
- 每个容器是否有明确端口，且端口冲突时能快速发现。
- compute 是否能上报指标，sink 是否能展示用户可理解的结果。
- 用户接入模式下，用户端容器是否可以手动启动，并通过页面看到任务编号、节点 IP、端口和结果。
- 自动化测评模式下，平台是否可以一键启动需要的 source/compute/sink 容器。

## 八、交付给本系统时请至少提供

1. 业务参数表。

3. 成功判定指标和结果展示样例。
4. 5-10 条自然语言输入样例。
5. 旧版本分支或 commit，以及旧版本如何跑通的命令。
