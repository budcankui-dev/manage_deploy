本文约定管理节点、计算节点和终端节点的固定目录、启动方式和验证命令。后续更新部署、排查节点资源、跑端到端验证时，以本文为准，避免临时目录、不同 Python 路径和旧进程造成环境漂移。

## 3. Node Agent 标准化更新

将本地 `node_agent/` 同步到每台被管理节点的固定目录：

```bash
for host in manage-compute-1 manage-compute-2 manage-compute-3; do
  ssh "$host" 'sudo mkdir -p /opt/manage_deploy/node_agent /var/log/manage_deploy'
  rsync -az --delete /Users/yanjia/codes/manage_deploy/node_agent/ "$host":/tmp/node_agent_sync/
  ssh "$host" 'sudo rsync -az --delete /tmp/node_agent_sync/ /opt/manage_deploy/node_agent/'
done
```

重启 Node Agent：

```bash
for host in manage-compute-1 manage-compute-2 manage-compute-3; do
  ssh "$host" '
    old=$(pgrep -f "uvicorn main:app --host 0.0.0.0 --port 8001" || true)
    if [ -n "$old" ]; then sudo kill $old || kill $old || true; sleep 1; fi
    cd /opt/manage_deploy/node_agent
    NVIDIA_SMI_BIN=/usr/bin/nvidia-smi \
    PYTHONPATH=/opt/manage_deploy/node_agent \
    nohup /usr/local/bin/python3.11 -m uvicorn main:app --host 0.0.0.0 --port 8001 \
      >/var/log/manage_deploy/node_agent.log 2>&1 < /dev/null &
    sleep 2
    curl -sS http://127.0.0.1:8001/health
    echo
    curl -sS http://127.0.0.1:8001/resources
    echo
  '
done
```

如果某台机器 `/usr/local/bin/python3.11` 不可用，应先固定安装路径或创建软链接，不要临时换成系统 Python 3.6/3.10 混跑。

## 5. Stash/VPN 与校园 IPv6

本机开启 Stash/VPN 时，需要放行校园数据面 IPv6，否则会出现 `No route to host`。

Stash 配置：

```text
跳过代理：2001:da8:215:6a01::/64
跳过路由：2001:da8:215:6a01::/64
```

验证：

```bash
route -n get -inet6 2001:da8:215:6a01:e381:3ba8:e0:c6c9
ping6 -c 2 2001:da8:215:6a01:e381:3ba8:e0:c6c9
```

如果 `route get` 不走 `en0`，或 `scutil --nwi` 显示 IPv6 不可达，先修 Stash/TUN 配置，不要误判为业务服务问题。

## 7. 批量业务测评回归

批量测评仍走平台自动控制 source/compute/sink 三容器，不等同于用户自启动模式。

回归顺序：

1. 系统设置中确认业务测评路由方式为本地 mock/内部自动路由。
2. 工作节点资源同步，确保 compute 节点 GPU 与 baseline 可见。
3. `/benchmark` 创建矩阵和视频各一小批测评。
4. 点击运行，确认成功率自动刷新。
5. 矩阵按 `effective_gflops >= baseline * 0.8` 判断。
6. 视频按 `frame_latency_p90_ms <= baseline * 1.5` 判断。

终端节点 h1-h13 不参与 baseline 测试，只作为 source/sink 端点或用户端演示节点。

## 8. 本次暴露的问题

- 管理节点 `/home/bupt/manage_deploy` 不是 git 仓库，不能直接 `git pull`，必须使用 rsync 或重新初始化标准部署仓库。
- 计算节点 Node Agent 曾从 `/tmp` 目录启动，更新文件后不一定作用到运行进程。
- Node Agent root 服务环境中可能找不到 `nvidia-smi`，导致 `/resources` 错误显示 GPU 为 0。代码已增加 `NVIDIA_SMI_BIN` 和常见绝对路径兜底，但部署时仍建议显式设置 `NVIDIA_SMI_BIN=/usr/bin/nvidia-smi`。
- compute-1/2 的 DB `business_ipv6` 曾是旧地址，导致用户端 source 按 `network_bindings` 访问 compute 时一直重试。节点 IPv6 变化后必须先刷新节点表，再跑 E2E。
