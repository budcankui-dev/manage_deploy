# 模态命名与意图解析扩展方案

本文档记录大课题中的模态命名，以及本系统在意图解析数据集和演示业务中的覆盖边界。

## 模态命名

用户提供的“基于模态定义的模态命名”材料中，按业务特征与资源需求归纳为 4 类场景、8 种模态。当前先完整记录如下；若后续验收口径明确只要求 7 种模态，可在数据集生成配置中裁剪其中一类。

| 场景 | 模态名称 | 典型应用 | 本系统中的覆盖方式 |
|------|----------|----------|--------------------|
| 大科学装置场景 | 高通量计算模态 | 高能物理科学实验 | 作为正式演示业务之一，对应矩阵乘法/科学计算任务，重点指标为计算吞吐量。 |
| 大科学装置场景 | 低时延转发模态 | AR/VR/XR 业务 | 作为正式演示业务之一，对应视频 AI 推理/工业检测抽帧链路，重点指标为帧推理 P90 时延。 |
| 大科学装置场景 | 智算中心模态 | 智算中心 | 作为意图解析数据集扩展模态，构造大模型文本生成、文本模型训练等智能计算需求。 |
| 大科学装置场景 | 分布式存算模态 | 中国电信视联网 | 作为意图解析数据集扩展模态，构造多节点存储、就近计算、数据拉取类需求。 |
| 电网综合数据网络场景 | 大规模连接模态 | 虚拟电厂 | 作为意图解析数据集扩展模态，构造大量终端接入、并发连接、周期采集类需求。 |
| 电网综合数据网络场景 | 确定性转发模态 | 配电线路巡检 | 作为意图解析数据集扩展模态，构造确定性时延、抖动上限、稳定转发类需求。 |
| 电网综合数据网络场景 | 高能效边缘计算模态 | 输电线路巡检 | 作为意图解析数据集扩展模态，构造边缘侧轻量推理、低功耗、就近处理类需求。 |

## 当前演示边界

1. 系统端到端真实部署演示优先保证两个业务：高通量计算模态下的矩阵计算业务、低时延转发模态下的视频AI推理业务。
2. 其他模态暂不要求完整业务镜像和真实部署链路，只用于意图解析数据集扩展、页面标签展示和专家说明。
3. 意图解析结果、管理端评测样本、用户端工单、业务任务中心和验收测试页面都应展示“模态”标签，使专家能够看出自然语言需求被归入哪类模态。
4. 路由系统对接时继续使用已有 DAG 描述和 placements 结果。模态字段用于表达业务类型和验收口径，不替代外部路由模块的选路结果。

## 意图解析数据集扩展要求

1. 数据集样本应覆盖上述模态名称，建议直接覆盖 8 种模态；若最终验收只要求 7 种，则保证至少 7 种模态有有效样本。
2. 数据集中的任务类型可以多于系统真实可部署业务类型；不可部署业务在意图解析评测中只校验“意图类型、模态、源终端、目的终端、开始时间、结束时间、关键业务参数”等结构化抽取能力。
3. 演示可部署业务仍以矩阵计算和视频AI推理为主，避免为了数据集扩展引入过多未跑通业务。
4. 每条样本的期望结果应包含 `modality` 字段，用中文模态名作为评测标准值。
5. 对缺失字段样本，仍应校验缺失字段集合是否正确；模态字段如果能从文本中明确判断，也应参与正确性评分。
6. 数据集覆盖四类路由策略：默认无倾向对应 `resource_guarantee`；用户表达“更快、高性能、尽早完成、低时延”等倾向对应 `fastest_completion`；用户表达“空闲、低负载、避开高负载、竞争少、均衡”等倾向对应 `load_balance`；用户表达“成本更低、省钱、少占资源、经济型”等倾向对应 `cost_priority`。
7. 数据集不应只靠固定句式堆叠数量。每类业务至少准备多种同语义改写模板，并让路由倾向以自然语言偏好出现，而不是要求用户直接输入策略枚举值。

## 页面与功能要求

1. 用户端意图解析结果卡片展示“业务类型”和“所属模态”。
2. 管理端意图解析参数准确率页面在样本列表、样本详情、单条检测结果中展示“所属模态”。
3. 业务任务中心和工单详情展示“所属模态”，方便专家从工单追溯到模态场景。
4. 意图评测页面支持选择固定数据集中的单个样本作为输入，运行单条解析检测，并展示输入文本、期望结果、实际解析结果、字段判定和是否正确。
5. 单条样本检测用于演示和排错，不替代正式 360 条固定数据集准确率评测。

## 路由 DAG 资源估计原则

1. 意图解析确认后，系统会生成外部路由系统接收的 DAG JSON；DAG 的 `nodes[].resources` 使用任务类型和 `data_profile` 的估计值，不要求用户手工填写。
2. 当前估计值采用“够用且不明显浪费”的验收演示口径：矩阵计算和视频推理的 compute 节点默认需要 1 个 GPU，source/sink 默认不需要 GPU。
3. DAG 的 `edges[]` 同时包含 `data_mb` 和 `bandwidth_mbps`，用于向外部路由系统表达链路数据量和带宽需求。该值是根据业务参数的保守估计，后续可替换为历史实测基准。
4. 外部路由系统最终仍以 placements 写回每个子任务的部署节点和 GPU 编号；本系统接收结果后负责物化实例、注入运行环境并采集业务指标。
